MechanicalTask
Cloud Toolkit for experimentating crowdsourcing projects using a micro(stupid)-task driven approach.

No need to deploy a server or a database to create your own micro-task platform. Build your project by creating or remixing existing micro-Task template code (html/javascript) and embed it on your own web page.

Background

This cloud toolkit is an attempt to democratize the development and experimentation of microtasking projects to anyone having some basic knowledge in html/javascript. It is based on two main principles.

  • Beyond monolithic projects: embeddable micro-tasks

    MechanicalTask avoids the need to deploy a server and a database to prototype a crowdsourcing project. But MechanicalTask is also not a platform to host projects. Actually it generates web widgets you can embed in your own website like a youtube video. So you host the project on your website (with further tutorials etc) without hosting its infrastructure. You can be creative and generate small web widgets used as a captcha for your website, or as an 'active' banner you can distribute to your partners' websites instead of advertising. You can also more easily create facebook apps

  • Coding in the cloud You can easily modify your web widget on the fly with an online editor. a shortcut of the traditional: local update of code in the development environment + ; modify it; redeploy the new version.

  • Openess: toward a micro-tasking ecosystem

    First the raw data generated by the participants is freely accessible, so any third party can aggregate/analyze

    Secondly We think that what is valuable is not only the data but also the mechanism you used to collect them, engage people. So if you generate a project via this platform your html/script template will be automatically shared with the community so other people can transform or combine it. We think such openess will facilitate the emergence of best practices in micro tasking, and leverage more quickly a potential community.

-NM

How does it work?

MechanicalTask is a distributed infrastructure using google fusion tables to store your tasks and the answers of volunteers. We use google fusion and not an internal database because it easily enables third parties to view, export, visualize or event insert dynamicall new tasks via a user interface or an API. MechanicalTask index a first fusion table used as a stream of micro-tasks to process, manages the allocation of tasks (with a given redundancy) to a pool of users and send the results to another fusion table (output) we generated. You are only in charge of the user interface to collect the data via a HTML/javascript template. Some helpers/libraries are available. We will use this template to generate the web widget. Once embedded on a website, visitors can start playing and completing some tasks. Note Google fusion has some constraints you need to be aware.

Images classification

A common toy example is to classify a large amount of photos. In this example we assume you have csv or xsl spreadsheets with all the url of the images to classify. Each row is a task with for instance the url of the image to classify. Once this table imported on a google fusion table (you can also have a programm filling/updating the fusion table to add new tasks/images), you need to add the ID of the google fusion and create the UI Template to generate the web widget and the output table.

  • Tasks input table with a list of images and other contextual information

  • Generated web widget to classify images.

A similar example is about annotating large amount of images e.g. tagging galaxies like GalaxyZoo. In this case the input table is the same as for the previous example about classifying images. Only the instructions , and tools to process the task change. So only the UI template needs to be updated.

Analyzing a large area

Let assume you want to create quickly a crowdmapping project asking to volunteers to identify objects inside a large area.
  • First you need to create a fusion table representing the micro-tasks to process. You can generate this table yourself. we also propose you a task generator cutting your map into a grid of smaller areas, easier to analyze individually. The result is sent to a google fusion table. You need to set up the permission of the fusion table so it is publicly visible if it is not already done.
  • >Then you need create a crowdsourcing project. first add the encrypted ID of the fusion table so MechanicalTask can index the table (without understanding that is about mapping).
  • As you are responsible of the interpretation of the tasks e.g the UI and the information you want to collect, you also must add a script template. This template can be copied and modified from other projects. This will generate a web widget and another fusion table to store the answers of volunteers.
  • Finally you can embed the web widget in your website and add further information (e.g. tutorial) if required.
  • At any time you have visualize and export the answers via the google fusion.

  • Task generator helping you to divide the analysis of a large area

  • GeoVisualisation (using the google fusion tools) of the generated input google fusion table with a lots of small areas

  • Generated web widget you can embed on your web site

  • GeoVisualization (using the google fusion tools) of the Generated output fusion table storing the answers
