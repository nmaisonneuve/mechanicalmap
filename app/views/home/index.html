Background

What is volatileTask?
VolatileTask is a Cloud Toolkit to democratize the launch of projects enabling micro-contributions from web users, using basic knowledge in HTML5/javascript and that you can host on your own website like a youtube video. Zero installation or deployment of a server is required: You data +your code are stored/shared and clonable thanks to cloud infrastructures

What is a micro-contribution?
Micro-contributions are contributions of any kind that someone could quickly make remotely (or not) within a small amount of time. If we think about the diversity of contributions in participatory projects as a long tail with few core people and a large majority of volatile users,the goal of these micro-applications is to tap into the latter segment assuming that the sum of these micro-contributions could leverage a subtantial work.

What kind of micro-contributions?
e.g. analyzing a large flow of individual tweets, classifying a large volume of images, analyzing a large area on a map, segmenting very quickly a long video collaboratively, generating stories via the writing of individual sentences , or any kind of big data processable/generable more easily by volatile participants than the machine.

Features

1. Decentralised / in the cloud

We do not host server, database, community, forum etc. VolatileTask takes a lazy/minimalist approach by using the web eco-system: it uses external web services, and thus not hosting/maintaining these components. no notion of server, no database to install,no deployment process, no local stuff to setup. If you want a community+forum, create a Facebook page.

2. Embeddable like a youtube video and Spreadable over the web

volatileTask generates micro-apps embeddable on your own website like a youtube video. You are responsible to host any further tutorials/doc on your website. Once embedded, web visitors can directly play with the app, or also embed the app to their websites to support your action beyond traditional advertising.

3. Open App = Open Data + Open code

VolatileTask has an open policy: any app developed with this infrastructure is Open. You data and your code are at any time accessible (read-only for non-admin). We use google fusion tables to manage your data and not an internal database because anyone can easilyview, export, analyze, visualize the raw data. Idem with your code via Github.

Online coding

You are in charge to code the UI to collect the data via an HTML/javascript template. VolatileTask enables to edit your code online and update your app on the fly. This shortcuts the traditional redeployment cycle for a faster iterative protoptyping. We also automatically save your code into the Gist/Github platform to manage versioning.

Clone existing apps

Anyone can clone existing micro-apps and modify the data or the code to generate their own variant crowdsourcing projects in few minutes. Such feature facilitates the emergence of eco-system with best practices.

Interaction design: from micro to macro problem solving

by default we use a parallel worfklow with redundancy mechanism. but you are free to design your own workflow e.g.iterative model, or inject the answer of a micro-app as input for other apps creating interaction between apps (TODO add section: "From User to Designer" about crowdsourcing/self-organisation the decomposition/design of tasks to solve a pb)

Mobile compatible

For each app, we generated the associated QRCode. displaying in full screen mode the micro-app.

Mechanical Turk compatible

YOu can use the Amazon mechanical turk service as fast and chip crowdsourcing simulator to test the all process or improve the design/usability via the input of the users. You need to inject a small code into your app
