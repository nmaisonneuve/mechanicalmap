MechanicalTask
Platform for experimentating crowdsourcing projects using a micro-task driven approach.

No need to deploy a server or a database to create your own micro-task platform. Build your project by creating or remixing existing micro-Task template code (html/javascript) and embed it on your own web page.

Background

This cloud toolkit is an attempt to democratize the development and experimentation of microtasking projects to anyone having some basic knowledge about html/javascript. It is based on two main principles.

  • Zero installation

    We live in the age of cloud computing so no need to set up your own server and a database. Our goal is also *not* to host crowdsourcing projects since we generate web widgets you can embed in your own website. Actually we are like a search engine maintaining an index of available tasks for volunteers. We index your tasks via a google fusion table used as a stream of input tasks. We manage the allocation of tasks and send the results to another fusion table (output) we generated. You are only in charge of the user interface to collect the data via a HTML/javascript template. Some helpers/libraries are available. We will use this template to generate the web widget . embeddable like a youtube video. You can also use it as a 'captcha with purpose', or as new kind of banner 'with a purpose' you can distribute to your partners' website instead of advertising.

  • Openess: toward a micro-tasking ecosystem

    We think that what is valuable is not only the data but also the mechanism you used to collect them. So if you generate a project via this platform your html/script template will be automatically shared with the community so other people can transform or combine it. We think such openess will facilitate the emergence of best practices in micro tasking, and leverage more quickly a potential community.

Analyzing a large area

Let assume you want to create quickly a crowdmapping project asking to volunteers to identify objects inside a large area.
  • First you need to create a fusion table representing the micro-tasks to process. You can generate this table yourself For crowdmapping projects, we propose you a task generator cutting your map into a grid of smaller areas, easier to analyze individually. The result is sent to a google fusion table.
  • >Then you need to create a project. you need to add the encrypted ID of the fusion table containing the task so the system could index the tasks. You need to set up the permission of the fusion table so it is public (without understanding that is about mapping).
  • As you are responsible of the interpretation of the tasks e.g the UI and the information you want to collect, you also must add a script template. This template can be copied and modified from other projects. This will generate a web widget and another fusion table to store the answers of volunteers.
  • Finally you can embed the web widget in your website and add further information (e.g. tutorial) if required.
  • At any time you have visualize and export the answers via the google fusion.

  • Task generator helping you to divide the analysis of a large area

  • GeoVisualisation (using the google fusion tools) of the generated input google fusion table with a lots of small areas

  • Generated web widget you can embed on your web site

  • GeoVisualization (using the google fusion tools) of the Generated output fusion table storing the answers

Images classification

Another common example is to classify a large amount of photos. In this example we assume you have csv or xsl spreadsheets with all the url of the images to classify. Each row is a task with for instance the url of the image to classify. Once this table imported on a google fusion table (you can also have a programm filling/updating the fusion table to add new tasks/images), you need to add the ID of the google fusion and create the UI Template to generate the web widget and the output table.

  • Tasks input table with a list of images and other contextual information

  • Generated web widget to classify images.

Tagging Images

A similar example is to annotate large amount of images (tag galaxies). In this case the tasks input table is the same as classifying images. Only the instructions and tools to do the task change. So only the UI template needs to be updated.
