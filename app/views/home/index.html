MechanicalTask
Platform for experimentating crowdsourcing projects using a micro-task driven approach.

No need to deploy a server or a database to create your own micro-task platform. Build your project by creating or remixing existing micro-Task template code (html/javascript) and embed it on your own web page.

Background

This cloud toolkit is an attempt to democratize the development and experimentation of microtasking projects to anyone having some basic knowledge about html/javascript. It is based on two main principles.

  • Zero server

    We live in the age of cloud computing so no need to set up your own server and a database. Our goal is also *not* to host projects since we generate web widgets you can embed in your website. Actually we are like a search engine maintaining an index of tasks for volunteers. We index your tasks via a google fusion table used a stream of input tasks. We manage the allocation of tasks and send the results to another fusion table (output) we generated.

    You are only in charge of the user interface to collect the data via a HTML/javascript template. Some helpers/libraries are available. From this script, we generate a web widget you can embed on your web page like a youtube video. (you can also used it as a 'captcha with purpose', or as new kind of banner replacing advertising on your partners' web sites ).

  • Openess: toward a micro-tasking ecosystem

    We think that what is valuable is not only the data but also the mechanism you used to collect them. So if you generate a project via this platform your html/script template will be automatically shared with the community so other people can transform or combine it. We think such openess will facilitate the emergence of best practices in micro tasking, and leverage more quickly a potential community.

Examples

Crowdmapping

Let assume you want to create quickly a crowdmapping project asking to volunteers to identify objects in a large area.
  • To achieve that , we propose you a task generator to cut your own map into a grid of smaller areas, easier to analyze individually. The result is a google fusion table
  • Create a new project and add the ID of this fusion table so the system will index the tasks (without understanding that is about mapping).
  • As you are responsible of the interpretation of the task e.g the UI and the information you want to collect, you must add a script template (that you can copy and transform from another project). This will generate a web widget and also an output fusion table to store the answers of volunteers.
  • Finally publish/embed the web widget in your website and add more information (tutorial) if required.
  • Analyze the answers by playing the google fusion table tools or export the data.

  • Task generator helping you to divide the analysis of a large area

  • Tasks input table(generated fusion table) with a grid of smaller areas. (1/2)

  • Visualisation of the tasks inputs. (2/2)

  • To create a new project, add the ID of the fusion table and a script template

  • Generated web widget you can embed on your web site

  • Generated output fusion table to store the answers
