Background

What is volatileTask?
VolatileTask is a Cloud Toolkit democratizing the building of participatory apps that leverage micro-contributions from web / mobile users for lightweight programmers by cloning apps and coding online. Zero installation is required (no server+database to deploy) thanks to Cloud infrastructures. Embed it on your own web site like a youtube video. Such infrastructure and its open policy (open app = open data + open code) will facilitate the emergence of an eco-system of micro-apps with best practices and conventions

What is a micro-contribution?
we live in a period of continous partial attention. Micro-contributions are contributions of any kind that someone could quickly make remotely (or not) within a small amount of time. If we think about the diversity of contributions in participatory projects as a long tail with few core people and a large majority of volatile users,the goal of these micro-applications is to tap into the latter segment assuming that the sum of these micro-contributions could leverage a subtantial work (but different from what hard-core contributors will produce).

Features

Minimal setup

No server + database to install, no deployment process, no code editor to install. VolatileTask has a green/minimalist approach by using web eco-system i.e. existing (cloud) services . If you want a community+forum for your app, create a Facebook page or a Google group

Embeddable and spreadable over the web

Embed any generated app on your web site like a Youtube video. Once embedded, web visitors can directly play with the app, or also embed the app to their websites to support your action beyond traditional advertising.

Open App = Open Data + Open code

VolatileTask has an open policy: any app developed with this infrastructure is open. You data and your code are at any time accessible by anyone (read-only for non-admin). We use google fusion tables to store your data and not an internal database because anyone can easily view, export, analyze, visualize this public raw data. Idem with your code via Github.

Cloud coding:

Code the task logic anywhere, anytime (why not from your mobile phone) by directly using an online editor coding online and update your app on the fly. This shortcuts the deployment cycle to get faster protoptyping. We also manage code versioning by using Gist/Github platform.

Clone apps

Anyone can clone existing apps and modify the data or the code to generate their own variant crowdsourcing projects in few minutes. For instance you can experiment different protocol variations and launch several cloned apps.

Workflow design

by default we use a parallel worfklow with redundancy mechanism. but you are free to design your own workflow e.g.iterative model, or inject the answer of a micro-app as input for other apps creating interaction between apps (TODO add section: "From Solver to Designer" about crowdsourcing/self-organisation the decomposition/design of tasks to solve a pb)

Mobile Compatible

Code an app directly from your mobile thanks to the online editor. We also generated the associated QRCode stick to access to app in full screen mode, so you can print it to advertise your app in the physical world.

Mechanical Turk compatible

You can use the Amazon mechanical turk service as fast and chip simulator to test your app or get tasks done by turkers. You need to inject a small code into your app
